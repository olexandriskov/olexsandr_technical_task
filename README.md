### **Willing to improve**

Працюючи над поставленими завданнями, виділив кілька аспектів поза межами ТЗ, які варто було б реалізувати, для наближення застосунку до cтатусу production-ready.

Зараз у DTO, сутності та вхідному скрипті БД використовується тип Double. Через потенційні проблеми з точністю чисел із плаваючою комою, цей тип не є бажаним для фінансів. У подальшому я б здійснив рефакторинг із переходом на BigDecimal та тип decimal у MySQL для гарантії точності розрахунків.

Для автоматизації змін схеми БД варто інтегрувати Flyway або Liquibase. Наразі використовується режим “ddl-auto: update”, чого для етапу розробки достатньо.

Побачив необхідність у створенні кастомних анотацій для перевірки розширень вхідних файлів. Це дозволить на етапі запиту відсікати шкідливі або некоректні типи контенту і запобігати завантаженню файлів, що можуть бути потенційно шкідливими.

Наразі файли зберігаються в локальній файловій системі. У реальному проєкті, я б замінив її на хмарне сховище, на кшталт AWS S3.

### Інструкція із запуску

1. Створіть базу даних MySQL з назвою fishstore.
2. Налаштуйте параметри доступу (username/password) у файлі src/main/resources/application.properties.
3. Запустіть проєкт.
4. Доступ адміністратора: логін “admin”, пароль “admin”; доступ користувача: логін “user”, пароль “user”;
